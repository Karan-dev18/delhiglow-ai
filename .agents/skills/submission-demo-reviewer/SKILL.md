# Submission Demo Reviewer

Run a final evidence-based check of the product, repository, deployment, and presentation. Treat anything that can derail the live demo as urgent.

## Audit Checklist

1. Run the production build and record the result.
2. Visit every route and test primary navigation, deep links, refreshes, and not-found behavior.
3. Complete the main user journey on mobile and desktop.
4. Exercise every visible AI feature with live configuration and missing-key fallback mode.
5. Confirm loading, empty, error, and recovery states.
6. Verify the live link, repository link, screenshots, and submission metadata.
7. Check that the README covers setup, features, AI workflow, fallback mode, architecture, and demo steps.
8. Map visible evidence to Product Thinking, UI/UX Design, AI Usage & Innovation, Execution Quality, and User Experience.
9. Rehearse a short demo script with a reliable reset path and backup screenshots.

## Reporting

Report findings in this order:

- **Submission blockers:** Build failures, broken routes, unavailable links, missing instructions, exposed secrets, or a failed core journey.
- **Demo risks:** Network dependencies, slow steps, fragile data, unclear AI behavior, or inconsistent fallback output.
- **Judging gaps:** Criteria without clear product evidence.
- **Polish opportunities:** Improvements that fit the remaining time.

Include concrete evidence for each finding and finish with a go/no-go recommendation.
