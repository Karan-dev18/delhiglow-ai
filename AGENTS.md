# DelhiGlow AI

## Mission

Build a winning AI-powered Delhi beauty salon marketplace for the AI Startup Buildathon 2026 – Beauty Salon Marketplace Challenge.

## Judging Criteria

Optimize every product and engineering decision for:

- Product Thinking
- UI/UX Design
- AI Usage & Innovation
- Execution Quality
- User Experience

## Tech Stack

- React
- Vite
- Tailwind CSS
- React Router
- Firebase (optional)
- Vercel

## Product Rules

- Keep the marketplace city-specific to Delhi, including realistic Delhi localities, needs, and salon discovery flows.
- Present DelhiGlow AI as a credible startup product, not a generic hackathon template.
- Make AI features visible, understandable, and demonstrable in the UI.
- Prefer a focused, polished end-to-end journey over a broad collection of incomplete features.

## Code Rules

- Build clean, focused components with clear responsibilities.
- Keep shared UI and logic in reusable files.
- Make every user-facing flow responsive across mobile and desktop.
- Include accessible labels, keyboard-friendly interactions, and sensible contrast.
- Provide loading, empty, and error states for data-driven experiences.
- Avoid unnecessary complexity and dependencies that put the demo at risk.

## AI Rules

- Include an AI Concierge for salon discovery and recommendations.
- Include AI-generated review summaries.
- Include a bridal beauty planner or timeline.
- Explain recommendations and outputs in user-friendly language.
- Provide deterministic fallback demo data whenever an AI API key is missing or a request fails.
- Never let an unavailable AI service block the core demo journey.

## Definition of Done

- `npm run build` passes.
- No routes or navigation links are broken.
- Core journeys work on mobile and desktop.
- AI features are visible and work with live AI or fallback demo mode.
- The README explains the AI workflow, fallback behavior, setup, and demo path.
